## 💡 调试建议

### 高效调试流程

```
1. 【定位问题】
   查看错误信息和堆栈跟踪
   找到可能出错的代码行

2. 【添加断点】
   在关键位置添加断点
   - 入口函数
   - 条件分支
   - 循环体

3. 【单步跟踪】
   F10: 跳过函数调用
   F11: 进入函数内部
   检查每一步的执行结果

4. 【观察变量】
   在变量面板查看状态
   或在调试控制台计算表达式
   
5. 【定位根因】
   通过步进，找到变量异常的位置
   理解逻辑错误原因

6. 【修复和验证】
   修改代码
   再次运行验证修复
```

### 常用 IDE 快捷键

**VS Code:**
- F5: 启动/继续调试
- F10: 单步跳过 (Step Over)
- F11: 单步进入 (Step Into)
- Shift+F11: 单步跳出 (Step Out)
- Ctrl+Shift+D: 打开调试器

**PyCharm:**
- Shift+F9: 启动/继续调试
- F8: 单步跳过
- F7: 单步进入
- Shift+F8: 单步跳出
- Alt+F9: 运行到光标

**Python pdb:**
```
n (next)     - 下一行

c (continue) - 继续执行
l (list)     - 列出代码
p (print)    - 打印变量
```

### 常见问题

**Q: 同步模式会影响性能吗?**
A: 不会。FastAPI 在同步函数上使用线程池执行，性能相同。

**Q: 如何调试 API 路由?**
A: 在路由函数设置断点，使用 API 客户端（curl/Postman）发送请求，IDE 会暂停在断点处。

**Q: 如何监看某个变量?**
A: 在 VS Code 的"监视"面板添加表达式，或在 pdb 中使用 `p 变量名` 命令。

**Q: 如何设置条件断点?**
A: 在断点上右键 → "编辑断点" → 输入条件（如 `i > 5`）。
